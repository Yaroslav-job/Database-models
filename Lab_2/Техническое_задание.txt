1.6. Назначение разработки
ИС ЖОС предназначена для автоматизации процессов:
	• учёта и хранения данных об успеваемости студентов;
	• ведения электронных журналов оценок и посещаемости преподавателями;
	• формирования отчётности и ведомостей для деканатов;
	• предоставления студентам и администрации онлайн-доступа к данным об обучении.
Система должна интегрироваться с другими ИС вуза (LMS, кадровая, бухгалтерская и др.) и поддерживать безопасный веб-доступ из любых современных браузеров.
1.7. Цели создания системы
	• Централизация и унификация данных об успеваемости.
	• Минимизация ручного труда преподавателей и сотрудников деканата.
	• Повышение прозрачности и достоверности учётных данных.
	• Автоматизация формирования отчётности по различным срезам.
	• Повышение цифровой зрелости учебного заведения.
	• Внедрение электронного документооборота и отказ от бумажных ведомостей.
1.8. Ожидаемые результаты внедрения
	• Сокращение времени обработки данных об успеваемости минимум на 40 %.
	• Исключение дублирования информации в разных отделах.
	• Возможность удалённого контроля прогресса студентов.
	• Повышение удовлетворённости пользователей (преподавателей, студентов, администрации).
________________________________________
1.9. Требования к видам обеспечения
Программное обеспечение
	• ОС сервера: Windows Server 2019 / Linux Ubuntu 20.04 LTS;
	• СУБД: PostgreSQL 12+ или аналог с поддержкой ACID;
	• Язык: Python 3.10+, фреймворк Django / FastAPI;
	• Веб-клиент: React / Vue.js;
	• API: REST + GraphQL;
	• Система логирования: ELK-stack или аналог;
	• Автоматизация: Docker + GitLab CI/CD;
	• Протокол связи: HTTPS (SSL/TLS 1.2+).
Информационное обеспечение
	• Реляционная структура БД с нормализацией до 3 НФ.
	• Поддержка импорта/экспорта JSON, CSV, XML.
	• Идентификация студентов по единому ID (GUID / UUID).
	• Обеспечение ведения исторических данных (versioning).
	• Синхронизация с LMS или ERP через API.
Техническое обеспечение
	• Сервер: CPU ≥ 8 ядер, RAM ≥ 16 ГБ, SSD ≥ 500 ГБ, сеть ≥ 1 Гбит/с.
	• Клиент: ПК/ноутбук, RAM ≥ 4 ГБ, современный браузер (Chrome, Edge, Firefox).
	• Резервирование: ежедневное бэкап-копирование БД с хранением копий ≥ 30 дней.
Организационное обеспечение
	• Обучение пользователей (преподавателей, администраторов, студентов).
	• Разработка регламентов эксплуатации и защиты данных.
	• Назначение ответственных за сопровождение ИС.
Методическое обеспечение
	• Методические рекомендации по работе в ИС.
	• Регламент обновления и ведения данных (частота, ответственные).
	• Инструкции по работе с отчётами и формированию ведомостей.

2. ХАРАКТЕРИСТИКИ ОБЪЕКТА АВТОМАТИЗАЦИИ
2.1. Общие сведения
Объект — процессы учёта и хранения информации об успеваемости студентов, включая дисциплины, учебные планы, посещаемость, экзаменационные ведомости, а также формирование отчётности.
2.2. Подробное описание функций
1.Управление учебными планами
	- Создание, редактирование, архивирование планов.
	- Автоматическое связывание дисциплин с кафедрами и преподавателями.
2.Управление дисциплинами
	- Привязка к учебному плану, установка кредитов, семестров.
	- Возможность загрузки РПД (рабочих программ).
3.	Управление пользователями
	- Добавление преподавателей, студентов, деканов, администраторов.
	- Импорт из LDAP/Active Directory.
4.	Учёт оценок и посещаемости
	- Ввод баллов за модули, практики, экзамены.
	- Настройка весовых коэффициентов.
	- Автоматический расчёт среднего балла и рейтинга.
5.	Формирование отчётности
	- Ведомости, рейтинги, графики успеваемости.
	- Выгрузка в PDF, Excel, CSV.
6.	Интеграция и обмен данными
	- Взаимодействие с LMS (Moodle, Canvas), API 1С, Excel-импорт.
7.	Аналитика и статистика
	- Дашборды по факультетам, курсам, дисциплинам.
	- Выявление проблемных студентов, анализ динамики.
8.	Администрирование
	- Настройка прав, резервирование данных, аудит действий.
2.3. Входные и выходные данные
(добавлено описание структур, форматов, частоты обновления)
Тип данных	Источник	Формат	Частота обновления
Учебный план	Администратор	JSON/DB	Ежегодно

Оценки	Преподаватель	Ввод через UI	Постоянно

Отчёты	Система	PDF/Excel	По запросу

3. ТРЕБОВАНИЯ К СИСТЕМЕ
3.1. Функциональные требования
	• Регистрация и авторизация с поддержкой MFA.
	• Роли: администратор, преподаватель, студент, декан.
	• Уведомления (email, push) при новых оценках.
	• Расширенный поиск и фильтрация по дисциплинам, датам, ФИО.
	• История изменений (версионность записей).
	• Возможность массового импорта/экспорта данных.
	• Гибкие шаблоны отчётов и графиков.
	• Интеграция с LMS и внешними БД через REST API.
	• Модуль обратной связи (обращения пользователей).
	• Поддержка календаря занятий и расписания.
3.2. Требования к надёжности
	• Дублирование данных в горячем резерве.
	• Автоматическое бэкап-копирование не реже 1 раза в сутки.
	• Среднее время восстановления ≤ 2 часов.
	• Средняя доступность (uptime) ≥ 99,5 %.
3.3. Требования к безопасности
	• Шифрование данных при передаче (HTTPS / SSL).
	• Хранение паролей с использованием bcrypt/Argon2.
	• Ролевая модель доступа (RBAC).
	• Логирование всех действий пользователей.
	• Контроль сессий и ограничение по IP/времени.
	• Соответствие требованиям 152-ФЗ и ПП 1119.
	• Возможность экспорта журналов аудита для ФСТЭК-проверки.
3.4. Требования к производительности
	• Среднее время ответа ≤ 2 сек.
	• Поддержка ≥ 500 одновременных пользователей.
	• Возможность горизонтального масштабирования (кластеризация).
3.5. Требования к удобству (UX/UI)
	• Современный веб-интерфейс, адаптивная верстка.
	• Поддержка мобильных устройств (Android/iOS).
	• Возможность кастомизации цветовой темы, языка интерфейса.
	• Наличие контекстных подсказок, интерактивных справок.
	• Соответствие требованиям ГОСТ Р 52872-2012 (доступность ПО).
3.6. Масштабируемость и расширяемость
	• Возможность добавления новых модулей без модификации ядра.
	• Использование микросервисной архитектуры.
	• Возможность миграции БД при обновлениях без потери данных.
3.7. Переносимость
	• Поддержка работы на Linux и Windows-платформах.
	• Возможность развертывания в частном или облачном дата-центре.
3.8. Требования к документации
	• Руководство пользователя (для всех ролей).
	• Руководство администратора.
	• Техническое описание архитектуры.
	• Описание API и форматов данных.
	• Программа и методика испытаний.
	• Инструкция по установке и обновлению.
